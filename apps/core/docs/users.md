# User Management

The platform uses role-based permissions with three main user groups.

## User Groups

| Group | Description | Permissions |
|-------|-------------|-------------|
| **Support Seeker** | Users seeking peer support | Can browse and request sessions |
| **Peer** | Trained peer supporters | Can host sessions, manage availability |
| **Neuromancer** | Staff/admin users | Full access to all features |

## Granting Peer Status

To upgrade a user to Peer status:

1. Navigate to **Users** in Wagtail admin
2. Select the user
3. Add them to the "Peer" group
4. The user will receive an email notification about their new role

## Profile Management

User profiles include:

- **Biography** - Public description
- **Languages** - Spoken languages for sessions
- **Profile image** - Avatar/photo
- **Certificates** - Training certificates (for trained peers)
- **Notification preferences** - Email notification settings
- **Privacy settings** - Profile visibility options

## User Settings

Each user can configure:

### Notification Settings

- Email notifications for session requests
- Email notifications for approvals
- Email notifications for payments

### Privacy Settings

- Profile visibility
- Contact information display

### Peer-Specific Settings

Additional settings available for Peer users:

- Session filter defaults
- Availability calendar preferences
