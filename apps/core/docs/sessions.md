# Managing Sessions

The platform supports two types of sessions for peer support.

## Session Types

### Peer Sessions

One-on-one sessions between a peer supporter and a support seeker.

- Host sets availability slots
- Support seekers request specific time slots
- Host approves or declines requests
- Payment processed through Stripe Connect

### Group Sessions

Sessions with multiple attendees facilitated by a host.

- Fixed date and time
- Multiple attendees can join
- Host manages participant list
- Single price for all attendees

## Session Workflow

1. **Creation** - Hosts create sessions through the session creation form
2. **Publication** - Sessions must be published to be visible to support seekers
3. **Availability** - Peer sessions require availability slots to be set
4. **Requests** - Support seekers can request to book sessions
5. **Approval** - Hosts approve or decline session requests
6. **Payment** - Sessions with pricing require payment through Stripe
7. **Completion** - Sessions are marked complete after the scheduled time

## Managing Session Requests

Navigate to **Sessions** in the sidebar to view and manage:

- Pending requests
- Approved sessions
- Payment status
- Session history

## Session Settings

Configure session-related settings in **Settings > Stripe Settings** for:

- Stripe API keys
- Application fee percentage
- Currency options

And **Settings > Whereby Settings** for:

- Video conferencing API key
- Meeting link generation
